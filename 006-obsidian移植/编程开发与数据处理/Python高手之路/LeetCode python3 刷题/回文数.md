## 题目

给你一个整数 `x` ，如果 `x` 是一个回文整数，返回 `true` ；否则，返回 `false` 。

回文数是指正序（从左向右）和倒序（从右向左）读都是一样的整数。

- 例如，`121` 是回文，而 `123` 不是。

**示例 1：**

**输入：** x = 121
**输出：** true

**示例 2：**

**输入：** x = -121
**输出：** false
**解释：** 从左向右读, 为 -121 。 从右向左读, 为 121- 。因此它不是一个回文数。

**示例 3：**

**输入：** x = 10
**输出：** false
**解释：** 从右向左读, 为 01 。因此它不是一个回文数。

**提示：**

- `-231 <= x <= 231 - 1`

**进阶：** 你能不将整数转为字符串来解决这个问题吗？

## 代码

### 直接反转（一行代码）

```python
class Solution:
    def isPalindrome(self, x: int) -> bool:
        return str(x) == str(x)[::-1]
```

但是执行用时和内存的消耗都比较严重。

### 反转一半数字（避免字符串转换）

```python
class Solution:
    def isPalindrome(self, x: int) -> bool:
        # 负数和个位为0的非0数都不是回文数
        if x < 0 or (x % 10 == 0 and x != 0):
            return False
            
        reversed_half = 0
        while x > reversed_half:
            reversed_half = reversed_half * 10 + x % 10
            x //= 10
        
        # 当数字长度为奇数时，reversed_half 会比 x 多一位，需要去掉中间位
        
        return x == reversed_half or x == reversed_half // 10
```

- 时间 O(log₁₀ n)，即数字的位数
    
- 空间 O(1)

## 反思

难懂，特别是第二种方法的这种反转一半数字，原理是什么？复杂度又是如何计算的？

如果目标是一个偶数位数字，那么反转一半后的数字必然是和剩下的 x 相等的，不会小于 x，也不会大于它。

当目标是一个奇数位数字的时候，以五位数为例，反转的结果必然是一个三位数，三位数必然大于两位数，那么就用到了判断条件`x = reversed_half // 10`，去掉反转后数字的最后一位，也就是原本奇数位数字的中间的那个数字，自然就能判断反转的结果和剩下数字的大小是否相同。

时间复杂度和空间复杂度的概念还不懂。
