
# Windows CMD 命令全解教程（从入门到进阶）

## 前言

CMD（命令提示符）是Windows系统内置的命令行解释器，是Windows系统管理、自动化操作、文件处理的核心工具。本教程按照「**基础入门→日常操作→脚本开发→系统管理→进阶应用**」的路径，完整讲解你提供的所有CMD命令，每个命令均包含**核心作用、语法结构、参数详解、实用示例、练习建议**，新手可直接按顺序学习。

---

## 第一部分：基础入门与导航命令（新手必学）

本部分是CMD的入门核心，所有操作都基于这些命令，优先掌握。

### 1. HELP

- **核心作用**：CMD的内置“帮助字典”，查看所有命令列表、指定命令的详细用法，是学习CMD的核心工具。

- **基础语法**：

    ```Plain Text
    
    HELP [命令名]
    ```

- **参数详解**：

| 参数                                                   | 作用                         |
| ---------------------------------------------------- | -------------------------- |
| 无参数                                                  | 列出所有CMD内置命令及简要说明（即你截图中的列表） |
| `[命令名]`                                              | 指定要查看详细帮助的命令，如 `HELP COPY` |
| > 补充：`[命令名] /?` 是等价用法，和 `HELP 命令名` 输出完全一致，是更常用的快捷写法。 |                            |
- **实用示例**：

    ```Plain Text
    
    HELP          # 查看所有命令列表
    HELP COPY     # 查看COPY命令的完整帮助
    DIR /?        # 查看DIR命令的帮助（快捷写法）
    HELP HELP     # 查看HELP命令本身的帮助
    ```

- **练习建议**：

    1. 执行 `HELP` 命令，浏览完整命令列表，标记出3个你感兴趣的命令；

    2. 分别用 `HELP 命令名` 和 `命令名 /?` 两种方式查看帮助，对比输出差异；

    3. 尝试用 `HELP` 查看你之前用过的 `ECHO`、`CD` 命令的完整用法。

### 2. CLS

- **核心作用**：清空当前CMD窗口的所有屏幕内容，还原干净的命令输入界面。

- **基础语法**：

    ```Plain Text
    
    CLS
    ```

- **参数详解**：无任何参数，直接执行即可。

- **实用示例**：

    ```Plain Text
    
    # 执行完一堆命令后，清空屏幕
    DIR
    CD Desktop
    CLS  # 一键清空，回到干净界面
    ```

- **练习建议**：

    1. 连续执行3条不同的命令（如 `VER`、`DATE`、`TIME`），观察屏幕内容；

    2. 执行 `CLS` 命令，验证清空效果；

    3. 养成习惯：屏幕内容杂乱时，用 `CLS` 清空，避免干扰后续操作。

### 3. EXIT

- **核心作用**：退出当前CMD.EXE会话（关闭命令提示符窗口）。

- **基础语法**：

    ```Plain Text
    
    EXIT [/B] [退出码]
    ```

- **参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    直接关闭当前CMD窗口|
|    `/B`|    仅退出当前批处理脚本，不关闭CMD窗口（批处理脚本专用）|
|    `[退出码]`|    指定一个数字作为程序退出码，用于脚本中传递执行结果（0通常代表成功，非0代表失败）|
- **实用示例**：

    ```Plain Text
    
    EXIT          # 直接关闭CMD窗口
    EXIT /B 0     # 批处理脚本中，正常退出，返回成功码0
    EXIT /B 1     # 批处理脚本中，异常退出，返回错误码1
    ```

- **练习建议**：

    1. 打开CMD窗口，执行 `EXIT`，验证窗口是否关闭；

    2. 新建一个bat脚本，写入 `@echo off & echo 测试 & EXIT /B 0`，执行后观察CMD窗口是否保留；

    3. 修改脚本退出码为1，执行后用 `echo %errorlevel%` 查看退出码。

### 4. CD / CHDIR

- **核心作用**：CD是CHDIR的简写，用于**显示当前工作目录、切换工作目录**，是CMD中最基础的导航命令。

- **基础语法**：

    ```Plain Text
    
    CD [/D] [路径]
    CHDIR [/D] [路径]
    ```

- **参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示当前所在的完整目录路径|
|    `[路径]`|    要切换到的目标目录，支持相对路径和绝对路径|
|    `/D`|    强制跨盘符切换目录（默认CD不能直接跨盘符，需加/D参数）|
|    `..`|    特殊路径，代表「上一级目录」|
|    `\`|    特殊路径，代表「当前盘符的根目录」|
|    `.`|    特殊路径，代表「当前目录」|
- **实用示例**：

    ```Plain Text
    
    CD                  # 显示当前目录路径
    CD Desktop          # 进入当前目录下的Desktop子目录
    CD ..               # 返回上一级目录
    CD \                # 切换到当前盘符的根目录
    CD /D D:\Project    # 跨盘符切换，直接从C盘切到D盘的Project目录
    CD C:\Windows\System32  # 用绝对路径切换到指定目录
    ```

- **练习建议**：

    1. 打开CMD，用 `CD` 查看当前目录，确认你的用户目录；

    2. 用相对路径切换到桌面目录，再用 `CD ..` 返回用户目录；

    3. 用绝对路径切换到 `C:\Windows` 目录，再用 `CD \` 回到C盘根目录；

    4. 尝试用 `/D` 参数跨盘符切换（如有D盘）。

### 5. DIR

- **核心作用**：列出当前目录/指定目录下的所有文件和子目录，是CMD中查看目录内容的核心命令。

- **基础语法**：

    ```Plain Text
    
    DIR [路径] [参数]
    ```

- **核心参数详解**：

| 参数   | 作用                                                  |
| ---- | --------------------------------------------------- |
| 无参数  | 列出当前目录的所有文件和文件夹，包含修改日期、大小等信息                        |
| `/P` | 分屏显示内容，内容过多时，按任意键才会显示下一屏，避免内容滚动太快                   |
| `/W` | 宽屏紧凑格式显示，仅显示文件名/文件夹名，一行显示多个，节省空间                    |
| `/S` | 递归显示当前目录+所有子目录的内容，可用于查找子目录中的文件                      |
| `/A` | 按属性筛选文件，如 `/A:H` 显示隐藏文件，`/A:D` 仅显示文件夹，`/A:-D` 仅显示文件 |
| `/B` | 极简模式，仅显示文件名/文件夹名，不显示日期、大小等额外信息                      |
| `/O` | 排序显示，如 `/O:N` 按名称排序，`/O:S` 按大小排序，`/O:D` 按修改日期排序     |
|      |                                                     |
- **实用示例**：

    ```Plain Text
    
    DIR                 # 列出当前目录的所有内容
    DIR /P              # 分屏显示目录内容
    DIR /W              # 紧凑宽屏显示
    DIR *.py            # 仅列出当前目录下所有.py后缀的文件（通配符用法）
    DIR /S /B *.py      # 递归查找当前目录及子目录下所有.py文件，仅显示路径
    DIR /A:D            # 仅显示当前目录下的所有文件夹
    DIR C:\Windows /O:S  # 列出C:\Windows目录的内容，按文件大小从小到大排序
    ```

| 参数      | 排序规则       | 效果         |
| ------- | ---------- | ---------- |
| `/O:N`  | 按名称升序（A→Z） | 文件名从 A 到 Z |
| `/O:-N` | 按名称降序（Z→A） | 文件名从 Z 到 A |
| `/O:S`  | 按大小升序（小→大） | 文件从小到大     |
| `/O:-S` | 按大小降序（大→小） | 文件从大到小     |
| `/O:D`  | 按日期升序（旧→新） | 时间从早到晚     |
| `/O:-D` | 按日期降序（新→旧） | 时间从晚到早     |

- **练习建议**：

    1. 进入桌面目录，执行 `DIR` 查看所有内容；

    2. 用 `DIR /A:D` 仅查看桌面的文件夹，用 `DIR /A:-D` 仅查看桌面的文件；

    3. 用 `DIR /S /B *.txt` 递归查找电脑中所有txt文件；

    4. 用 `DIR /O:D` 按修改日期排序查看文件，找到最新修改的文件。

- 直接按 **`Q` 键**（Quit 的缩写），**一键立即退出**，回到 CMD 提示符，无需按任意键翻完所有内容。

> 补充：`MORE` 命令的常用交互快捷键
> 
> - `Q`：立即退出（最常用）
> - 空格键：翻下一整屏
> - 回车键：翻下一行
> - `P`：输入行数，翻指定行数
> - `S`：跳过指定行数

| 快捷键        | 作用                | 适用场景                          |
| ---------- | ----------------- | ----------------------------- |
| `Ctrl + C` | 强制终止当前命令，回到提示符    | 所有命令卡死、长输出、持续运行命令（首选）         |
| `Q`        | 退出`MORE`分屏        | 仅对`more`分屏有效，对`dir`/`find`等无效 |
| `Ctrl + Z` | 退出交互式程序（如 Python） | 交互式 shell，对`dir`等无效           |
| `Alt + F4` | 强制关闭 CMD 窗口       | 极端卡死，`Ctrl + C`无效时            |

### 6. VER

- **核心作用**：显示当前Windows系统的版本号。

- **基础语法**：

    ```Plain Text
    
    VER
    ```

- **参数详解**：无参数，直接执行。

- **实用示例**：

    ```Plain Text
    
    VER  # 输出示例：Microsoft Windows [版本 10.0.22631.4317]
    ```

- **练习建议**：执行 `VER` 命令，记录你的Windows系统版本号。

### 7. TITLE

- **核心作用**：设置当前CMD会话的窗口标题，区分多个CMD窗口。

- **基础语法**：

    ```Plain Text
    
    TITLE [窗口标题]
    ```

- **参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    还原为CMD默认标题：`命令提示符`|
|    `[窗口标题]`|    自定义的窗口标题，支持中文、空格|
- **实用示例**：

    ```Plain Text
    
    TITLE Python开发环境  # 将窗口标题改为“Python开发环境”
    TITLE 批量文件处理    # 区分不同用途的CMD窗口
    TITLE                 # 还原默认标题
    ```

- **练习建议**：

    1. 打开2个CMD窗口，分别用 `TITLE` 设置不同的标题，观察窗口标题变化；

    2. 执行 `TITLE` 无参数命令，还原默认标题。

### 8. COLOR

- **核心作用**：设置CMD窗口的前景色（文字颜色）和背景色。

- **基础语法**：

    ```Plain Text
    
    COLOR [背景色][前景色]
    ```

- **参数详解**：

颜色用1位十六进制数表示，顺序是**先背景色，后前景色**，颜色对应表：

|数值|颜色|数值|颜色|
|---|---|---|---|
|0|黑色|8|灰色|
|1|蓝色|9|亮蓝色|
|2|绿色|A|亮绿色|
|3|湖蓝色|B|亮湖蓝色|
|4|红色|C|亮红色|
|5|紫色|D|亮紫色|
|6|黄色|E|亮黄色|
|7|白色|F|亮白色|
|> 注意：前景色和背景色不能是同一个值，否则命令无效；无参数执行 `COLOR` 会还原为CMD默认的黑底白字。||||
- **实用示例**：

    ```Plain Text
    
    COLOR 0A  # 黑底亮绿色文字（经典黑客风）
    COLOR F0  # 白底黑字
    COLOR 1F  # 蓝底亮白字
    COLOR     # 还原默认黑底白字
    ```

- **练习建议**：

    1. 尝试3种不同的配色方案，观察窗口效果；

    2. 执行 `COLOR` 还原默认配色；

    3. 尝试用相同的前景色和背景色，验证命令是否生效。

### 9. CMD

- **核心作用**：在当前CMD会话中，打开一个新的Windows命令解释器子进程/新窗口。

- 核心语法

```
cmd [/a | /u] [/q] [/d] [/e:on | /e:off] [/f:on | /f:off] [/v:on | /v:off] [[/s] [/c | /k] 命令] [/t:bf]
```

- 高频实用参数

| 参数      | 作用                                    | 示例                                               |
| ------- | ------------------------------------- | ------------------------------------------------ |
| `/k 命令` | 执行命令后**保留窗口**，新 CMD 窗口持续运行            | `cmd /k "python --version"` 打开新窗口，自动查看 Python 版本 |
| `/c 命令` | 执行命令后**自动关闭窗口**，适合脚本中临时执行             | `cmd /c "echo 测试 & pause"` 执行完自动关闭               |
| `/q`    | 关闭命令回显，等价于`@echo off`                 | `cmd /q` 新窗口默认不显示命令本身                            |
| `/t:bf` | 设置新窗口的背景色 (b) 和前景色 (f)，比如`/t:0A`黑底亮绿字 | `cmd /t:0A` 直接打开黑客风配色的新 CMD                      |
| `/s`    | 增强命令解析，处理带特殊字符的命令                     | `cmd /s /c "echo 1>2"` 正确输出`1>2`，避免重定向           |

| 参数      | 作用                            |
| ------- | ----------------------------- |
| 无参数     | 打开一个新的CMD子会话，执行 `EXIT` 可返回原会话 |
| `/C 命令` | 执行指定的命令，执行完成后自动关闭新的CMD进程      |
| `/K 命令` | 执行指定的命令，执行完成后保留新的CMD窗口（最常用）   |
| `/Q`    | 关闭回显，相当于执行了 `@echo off`       |
- **实用示例**：

    ```Plain Text
    
    CMD                 # 打开新的CMD子会话
    CMD /K "cd Desktop & dir"  # 打开新CMD窗口，自动切换到桌面并列出内容
    CMD /C "echo 测试 & pause" # 执行命令，执行完自动关闭窗口
    ```

- **练习建议**：

    1. 执行 `CMD /K "python --version"`，打开新窗口并自动查看Python版本；

    2. 执行 `CMD /C "echo 测试"`，观察窗口是否自动关闭；

    3. 执行 `CMD` 打开子会话，用 `EXIT` 返回原会话。

### 10. DOSKEY

- **核心作用**：编辑命令行、撤回历史命令、创建命令宏（快捷键），是提升CMD操作效率的核心工具。

- **基础语法**：

    ```Plain Text
    
    DOSKEY [宏名=命令] [参数]
    ```

- **核心参数详解**：

|    参数/功能|    作用|
|---|---|
|    上下方向键|    快速调出之前执行过的历史命令（默认开启）|
|    `/HISTORY`|    显示当前会话的所有命令历史|
|    `/MACROS`|    显示当前已创建的所有宏|
|    `宏名=命令`|    创建自定义宏，相当于给长命令设置快捷键|
|    `$*`|    宏中的通配符，代表宏执行时传入的所有参数|
- **实用示例**：

    ```Plain Text
    
    DOSKEY /HISTORY          # 查看当前会话的所有执行过的命令
    DOSKEY ls=DIR /B         # 创建宏：输入ls，等价于执行DIR /B
    DOSKEY cdpy=cd C:\PythonProject  # 创建宏：输入cdpy，一键切换到Python项目目录
    DOSKEY d=DIR $*          # 创建宏：d 等价于 DIR，支持传入参数，如 d /P
    DOSKEY /MACROS           # 查看所有已创建的宏
    ```

- **练习建议**：

    1. 执行 `DOSKEY /HISTORY`，查看你之前执行过的所有命令；

    2. 创建一个宏 `clear=CLS`，实现输入clear就清空屏幕；

    3. 创建一个宏 `py=python $*`，实现输入py就等价于python，支持传入文件名；

    4. 用 `DOSKEY /MACROS` 查看你创建的所有宏。

### 11. MORE

- **核心作用**：逐屏显示输出内容，解决长文本输出滚动太快的问题，常和管道符 `|` 搭配使用。

- **基础语法**：

    ```Plain Text
    
    MORE [文件名]
    命令 | MORE
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    从标准输入读取内容，逐屏显示，按空格键显示下一屏，按Enter键显示下一行，按Q键退出|
|    `/E`|    开启扩展功能，支持更多交互操作|
|    `/C`|    显示前清空屏幕|
- **实用示例**：

    ```Plain Text
    
    MORE 长文本.txt        # 逐屏查看长文本文件的内容
    DIR C:\Windows | MORE  # 逐屏显示C:\Windows目录的内容，避免滚动太快
    HELP COPY | MORE       # 逐屏查看COPY命令的帮助文档
    ```

- **练习建议**：

    1. 执行 `DIR C:\Windows\System32 | MORE`，体验逐屏显示效果；

    2. 用 `MORE` 查看一个你创建的py文件的内容；

    3. 尝试按空格键、Enter键、Q键，熟悉交互操作。

## 第二部分：文件与目录操作命令（日常核心）

本部分是CMD最常用的功能，用于文件/文件夹的创建、复制、移动、删除、重命名等操作，是日常使用的核心。

### 1. MD / MKDIR

- **核心作用**：MD是MKDIR的简写，用于创建新的目录（文件夹）。

- **基础语法**：

    ```Plain Text
    
    MD [路径]文件夹名
    MKDIR [路径]文件夹名
    ```

- **参数详解**：

|    参数|    作用|
|---|---|
|    `文件夹名`|    要创建的文件夹名称，支持相对路径和绝对路径|
|    扩展功能|    支持一次性创建多级目录（无需逐级创建），如 `MD a\b\c` 会直接创建a文件夹，a里的b，b里的c|
- **实用示例**：

    ```Plain Text
    
    MD src                # 在当前目录创建src文件夹
    MKDIR utils logs      # 一次性创建utils和logs两个文件夹
    MD C:\Users\SUN\Desktop\test  # 用绝对路径在桌面创建test文件夹
    MD project\src\main  # 一次性创建多级目录，无需先建project
    ```

- **练习建议**：

    1. 在桌面创建一个名为 `cmd_test` 的文件夹；

    2. 进入该文件夹，一次性创建 `src`、`backup`、`logs` 三个文件夹；

    3. 用一条命令创建多级目录 `project\code\python`；

    4. 用 `DIR` 验证所有文件夹是否创建成功。

### 2. RD / RMDIR

- **核心作用**：RD是RMDIR的简写，用于删除目录（文件夹）。

- **基础语法**：

    ```Plain Text
    
    RD [参数] [路径]文件夹名
    RMDIR [参数] [路径]文件夹名
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    仅能删除**空文件夹**，如果文件夹内有文件/子目录，会报错「目录不是空的」|
|    `/S`|    递归删除，删除指定文件夹及其所有子目录、文件（可删除非空文件夹）|
|    `/Q`|    静默模式，删除时不弹出确认提示，直接删除（高危参数，需谨慎）|
- **实用示例**：

    ```Plain Text
    
    RD empty_folder        # 删除空文件夹empty_folder
    RD /S test_folder      # 删除test_folder及其所有内容，会弹出确认提示
    RD /S /Q test_folder   # 强制静默删除test_folder及其所有内容，无提示（高危）
    ```

- **⚠️ 高危警告**：`RD /S /Q` 执行后，文件会被永久删除，不进入回收站，无法恢复！执行前务必确认路径正确！

- **练习建议**：

    1. 创建一个空文件夹 `test_empty`，用 `RD` 直接删除；

    2. 创建一个文件夹 `test_full`，在里面创建一个txt文件，尝试用无参数 `RD` 删除，观察报错；

    3. 用 `RD /S test_full` 删除该文件夹，体验确认流程；

    4. 再次创建 `test_full`，用 `RD /S /Q test_full` 静默删除，验证效果。

### 3. COPY

- **核心作用**：将一个或多个文件复制到另一个位置，是CMD最基础的文件复制命令。

- **基础语法**：

    ```Plain Text
    
    COPY [参数] 源文件 [目标路径/目标文件名]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/Y`|    覆盖目标文件时，不弹出确认提示，直接覆盖（批处理脚本常用）|
|    `/-Y`|    覆盖目标文件时，强制弹出确认提示（手动操作默认）|
|    `/A`|    以ASCII文本模式复制文件，用于文本文件|
|    `/B`|    以二进制模式复制文件，用于图片、exe、压缩包等非文本文件（默认模式）|
|    `/V`|    复制完成后校验文件完整性，确保写入正确，重要文件推荐使用|
|    `/Z`|    可重启模式复制网络文件，断网后支持断点续传|
- **实用示例**：

    ```Plain Text
    
    COPY src\main.py backup\                # 将main.py复制到backup目录，文件名不变
    COPY src\main.py src\main_backup.py     # 复制文件并重命名为main_backup.py
    COPY *.py backup\                        # 批量复制当前目录所有.py文件到backup目录
    COPY /Y src\main.py backup\              # 直接覆盖，不弹确认提示
    COPY /V src\重要文件.zip D:\备份\       # 复制后校验文件完整性
    COPY 1.txt+2.txt+3.txt 合并.txt          # 合并多个文本文件为一个文件
    ```

- **练习建议**：

    1. 创建一个 `hello.py` 文件，用 `COPY` 复制一份，命名为 `hello_backup.py`；

    2. 创建 `backup` 文件夹，用 `COPY` 将所有.py文件复制到该文件夹；

    3. 用 `COPY` 合并2个txt文件，验证合并效果；

    4. 用 `/V` 参数复制一个文件，体验校验功能。

### 4. XCOPY

- **核心作用**：COPY命令的增强版，支持复制文件、目录树、子目录，是批量复制文件夹的首选命令。

- **基础语法**：

    ```Plain Text
    
    XCOPY 源路径 目标路径 [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/E`|    复制所有子目录，包括空目录（最常用）|
|    `/S`|    复制所有子目录，不包括空目录|
|    `/I`|    如果目标路径不存在，自动创建目标目录（避免“是文件还是目录”的提示）|
|    `/Y`|    覆盖时不弹确认提示，直接覆盖|
|    `/H`|    复制隐藏文件和系统文件（默认COPY/XCOPY不复制隐藏文件）|
|    `/R`|    覆盖只读文件|
|    `/Z`|    可重启模式复制网络文件，断点续传|
- **实用示例**：

    ```Plain Text
    
    XCOPY src backup /E /I /H /Y  # 完整复制src目录的所有内容（含空目录、隐藏文件）到backup，自动创建目录，无提示覆盖
    XCOPY *.py D:\备份\python /I /Y  # 批量复制所有.py文件到D盘备份目录，自动创建目录
    XCOPY C:\项目 D:\备份\项目 /E /H /Z  # 备份整个项目目录，支持网络路径断点续传
    ```

- **练习建议**：

    1. 创建一个多级目录 `project\src\code`，里面放入几个py文件；

    2. 用 `XCOPY project project_backup /E /I /Y` 完整复制整个目录；

    3. 用 `DIR` 验证备份目录的内容是否和原目录一致；

    4. 尝试用 `XCOPY` 批量复制指定类型的文件到新目录。

### 5. ROBOCOPY

- **核心作用**：Windows内置的高级文件复制/备份工具，功能远超XCOPY，是Windows系统文件同步、备份的专业工具。

- **基础语法**：

    ```Plain Text
    
    ROBOCOPY 源目录 目标目录 [文件匹配规则] [参数]
    ```

- **核心参数详解**：

| 参数          | 作用                                          |
| ----------- | ------------------------------------------- |
| `/E`        | 复制所有子目录，包括空目录                               |
| `/MIR`      | 镜像模式，完全同步源目录和目标目录（目标目录中源目录没有的文件会被删除，用于增量备份） |
| `/Z`        | 可重启模式，断点续传，适合大文件/网络复制                       |
| `/R:n`      | 复制失败时重试n次，默认重试100万次                         |
| `/W:n`      | 重试间隔n秒，默认30秒                                |
| `/NP`       | 不显示复制进度，适合脚本中使用                             |
| `/LOG:文件路径` | 将复制日志写入指定文件，用于备份记录                          |
- **实用示例**：

    ```Plain Text

    ROBOCOPY src backup *.py *.txt /E  # 仅复制src目录下的.py和.txt文件，包括子目录
    ROBOCOPY C:\重要文件 D:\备份 /MIR /LOG:备份日志.txt  # 镜像备份，同时生成日志文件
    ```

- **练习建议**：

    1. 用 `ROBOCOPY` 镜像备份你的桌面文件夹到D盘的一个新目录；

    2. 尝试用 `ROBOCOPY` 仅复制指定类型的文件；

    3. 用 `/LOG` 参数生成复制日志，查看日志内容。

### 8. DEL / ERASE

- **核心作用**：DEL是ERASE的简写，用于删除一个或多个文件，**仅能删除文件，不能删除文件夹**。

### 10. ATTRIB

- **核心作用**：显示或修改文件/文件夹的属性（只读、隐藏、系统、存档等）。

### 11. COMP / FC

## 第三部分：批处理脚本核心命令（自动化必备）

本部分是编写 `.bat` 批处理脚本的核心命令，用于实现自动化操作、循环、条件判断、变量管理等，是CMD进阶的核心。

### 1. ECHO

- **核心作用**：显示消息、控制命令回显的开关，是批处理脚本中最基础的命令。

- **基础语法**：

    ```Plain Text
    
    ECHO [ON|OFF]
    ECHO 要显示的消息
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `ECHO ON`|    开启命令回显，执行脚本时，会显示每一条执行的命令（默认开启）|
|    `ECHO OFF`|    关闭命令回显，执行脚本时，只显示命令的输出结果，不显示命令本身|
|    `@ECHO OFF`|    关闭回显，同时连 `ECHO OFF` 这条命令本身也不显示，是批处理脚本的开头标配|
|    `ECHO.`|    输出一个空行，用于优化输出格式|
|    `ECHO 内容 > 文件`|    将内容覆盖写入文件|
|    `ECHO 内容 >> 文件`|    将内容追加写入文件末尾|
- **实用示例**：

    ```Plain Text
    
    @ECHO OFF          # 批处理脚本开头标配，关闭所有回显
    ECHO 程序开始执行
    ECHO.
    ECHO 正在处理文件...
    ECHO 处理完成！
    ```

### 5. CALL

- **核心作用**：从一个批处理脚本中调用另一个批处理脚本，执行完被调用的脚本后，返回原脚本继续执行。

- **基础语法**：

|    参数|    作用|
|---|---|

### 8. IF

- **核心作用**：批处理脚本中的条件判断命令，实现有条件的执行逻辑，是自动化脚本的核心。

- **基础语法**：

    ```Plain Text
    
    IF [条件] (
      条件成立时执行的命令
    ) ELSE (
      条件不成立时执行的命令
    )
    ```

- **核心条件类型**：

|    条件类型|    语法|    作用|
|---|---|---|
|    文件存在判断|    `IF EXIST 文件名`|    判断文件/文件夹是否存在|
|    字符串相等|    `IF "字符串1"=="字符串2"`|    判断两个字符串是否相等|
|    数值比较|    `IF 数字1 比较符 数字2`|    数值大小比较，比较符：EQU(等于)、NEQ(不等于)、LSS(小于)、LEQ(小于等于)、GTR(大于)、GEQ(大于等于)|
|    错误码判断|    `IF ERRORLEVEL 数字`|    判断上一条命令的退出码是否大于等于指定数字|
|    取反|    `IF NOT 条件`|    条件不成立时执行|
- **实用示例**：

    ```Plain Text
    
    @echo off
    # 1. 文件存在判断
    IF EXIST hello.py (
      echo hello.py文件存在
    ) ELSE (
      echo hello.py文件不存在，正在创建...
      echo print("hello") > hello.py
    )
    
    # 2. 数值比较
    SET /A num=10
    IF %num% GTR 5 (
      echo 数字大于5
    )
    
    # 3. 字符串判断
    SET /P input=请输入yes/no：
    IF /I "%input%"=="yes" (
      echo 你选择了yes
    ) ELSE (
      echo 你选择了no
    )
    ```

- **练习建议**：

    1. 编写一个脚本，判断当前目录是否存在 `backup` 文件夹，不存在就创建；

    2. 编写一个脚本，接收用户输入的数字，判断是奇数还是偶数；

    3. 编写一个脚本，判断上一条命令是否执行成功，给出对应的提示。

### 9. GOTO


- **练习建议**：

### 10. FOR

- **核心作用**：循环命令，为一组文件/数据中的每个元素执行指定的命令，是批量处理的核心。

- **基础语法**：

    ```Plain Text
    
    # CMD命令行中
    FOR %变量 IN (集合) DO 命令
    
    # 批处理脚本中（变量必须用双百分号）
    FOR %%变量 IN (集合) DO 命令
    ```

- **核心参数与用法**：

|    用法|    作用|
|---|---|
|    基础文件循环|    `FOR %%i IN (*.py) DO echo %%i`：遍历所有.py文件|
|    `/D`|    遍历目录，`FOR /D %%d IN (*) DO echo %%d`：遍历当前目录的所有文件夹|
|    `/R`|    递归遍历，`FOR /R %%i IN (*.py) DO echo %%i`：递归遍历所有子目录的.py文件|
|    `/L`|    数字序列循环，`FOR /L %%i IN (起始值,步长,结束值) DO 命令`：如 `FOR /L %%i IN (1,1,5) DO echo %%i` 输出1-5|
|    `/F`|    解析文件内容/命令输出，逐行处理，是最强大的循环用法|
- **实用示例**：

    ```Plain Text
    
    # 1. 批量重命名所有txt文件为md文件
    FOR %%i IN (*.txt) DO REN %%i %%~ni.md
    
    # 2. 批量删除所有pyc缓存文件
    FOR /R %%i IN (*.pyc) DO DEL %%i
    
    # 3. 循环输出1到10
    FOR /L %%i IN (1,1,10) DO echo 第%%i行
    
    # 4. 遍历当前目录的所有文件夹
    FOR /D %%d IN (*) DO echo 文件夹：%%d
    ```

- **练习建议**：

    1. 用 `FOR` 循环批量创建10个txt文件，命名为1.txt到10.txt；

    2. 用 `FOR` 循环批量将这些txt文件重命名为doc_1.txt到doc_10.txt；

    3. 用 `FOR /R` 递归查找当前目录所有的py文件，输出它们的完整路径。

### 11. SHIFT

- **核心作用**：调整批处理脚本中可替换参数的位置，用于处理脚本接收的多个传入参数。

- **基础语法**：

    ```Plain Text
    
    SHIFT [/n]
    ```

- **参数详解**：

批处理脚本的传入参数用 `%1`、`%2`、`%3`... 表示，`SHIFT` 会将参数整体左移一位，`%2` 变成 `%1`，`%3` 变成 `%2`，以此类推，用于处理不确定数量的参数。

- **实用示例**：

    ```Plain Text
    
    @echo off
    :loop
    IF "%1"=="" GOTO end
    echo 参数：%1
    SHIFT
    GOTO loop
    
    :end
    echo 所有参数处理完成
    PAUSE
    ```

    执行 `test.bat a b c d`，会依次输出a、b、c、d四个参数。

- **练习建议**：

    1. 编写一个脚本，用 `SHIFT` 处理传入的多个文件名，依次显示每个文件的内容；

    2. 执行脚本，传入多个文件名，验证处理效果。

### 12. PROMPT

- **核心作用**：更改CMD的命令提示符样式，自定义提示符的显示内容。

- **基础语法**：

    ```Plain Text
    
    PROMPT [自定义样式]
    ```

- **核心占位符**：

|    占位符|    作用|
|---|---|
|    `$P`|    当前完整路径|
|    `$G`|    大于号 `>`（默认提示符的结尾）|
|    `$T`|    当前时间|
|    `$D`|    当前日期|
|    `$N`|    当前盘符|
|    `$Q`|    等于号 `=`|
|    无参数|    还原为默认提示符 `$P$G`|
- **实用示例**：

    ```Plain Text
    
    PROMPT $P$G          # 还原默认提示符：路径+>
    PROMPT $T $G         # 提示符显示当前时间+>
    PROMPT $D $P $G      # 提示符显示日期+路径+>
    PROMPT SUN>           # 自定义提示符为SUN>
    ```

- **练习建议**：

    1. 尝试3种不同的提示符样式，观察CMD窗口的变化；

    2. 执行 `PROMPT` 无参数命令，还原默认样式。

## 第四部分：系统信息与管理命令（运维/排错）

本部分用于查看系统信息、管理进程、服务、磁盘、系统配置等，是Windows系统管理的核心命令。

### 1. SYSTEMINFO

- **核心作用**：显示计算机的详细系统配置信息，包括操作系统版本、硬件信息、补丁、网卡、内存、CPU等。

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示本机的完整系统信息|
|    `/S 计算机名`|    查看远程计算机的系统信息|
|    `/FO 格式`|    指定输出格式，支持 `TABLE`、`LIST`、`CSV`|
- **实用示例**：

    ```Plain Text
    
    SYSTEMINFO                    # 显示本机完整系统信息
    SYSTEMINFO | MORE             # 逐屏查看系统信息
    SYSTEMINFO | find "内存"      # 筛选查看内存相关信息

    ```

- **练习建议**：

    3. 将系统信息导出为CSV文件，用Excel打开查看。

### 2. TASKLIST

- **核心作用**：显示当前计算机中所有正在运行的进程，包括进程名、PID、内存占用、会话信息等。

- **基础语法**：

    ```Plain Text
    
    TASKLIST [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示所有运行中的进程|
|    `/FI 筛选器`|    按条件筛选进程，如按名称、PID、内存占用筛选|
|    `/M`|    显示每个进程加载的dll模块|
|    `/V`|    显示进程的详细信息，包括启动时间、用户、路径等|
|    `/FO 格式`|    指定输出格式，TABLE、LIST、CSV|
- **实用示例**：

    ```Plain Text
    
    TASKLIST                          # 显示所有进程
    TASKLIST | find "python.exe"     # 筛选查看所有Python进程
    TASKLIST /FI "MEMUSAGE gt 100000"  # 筛选内存占用大于100MB的进程
    TASKLIST /FI "IMAGENAME eq chrome.exe"  # 筛选所有Chrome进程
    TASKLIST /V /FO CSV > 进程列表.csv  # 导出详细进程列表到CSV
    ```

- **练习建议**：

    1. 执行 `TASKLIST`，查看当前运行的所有进程；

    2. 筛选查看你电脑上的Chrome、Python进程；

    3. 筛选查看内存占用大于50MB的进程。

### 3. TASKKILL

- **核心作用**：终止/停止正在运行的进程或应用程序，是结束卡死程序的核心工具。

- **基础语法**：

    ```Plain Text
    
    TASKKILL [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/IM 进程名`|    按进程名终止进程，如 `/IM python.exe`|
|    `/PID 进程ID`|    按PID进程号终止进程|
|    `/F`|    强制终止进程，针对卡死、无法正常关闭的程序（必加）|
|    `/T`|    终止指定进程及其所有子进程|
|    `/FI 筛选器`|    按条件批量终止进程|
- **⚠️ 警告**：强制终止系统进程可能导致系统崩溃，仅终止你熟悉的应用进程！

- **实用示例**：

    ```Plain Text
    
    TASKKILL /F /IM notepad.exe          # 强制关闭所有记事本进程
    TASKKILL /F /PID 12345               # 强制关闭PID为12345的进程
    TASKKILL /F /IM python.exe /T        # 强制关闭所有Python进程及其子进程
    TASKKILL /F /FI "MEMUSAGE gt 500000" # 强制关闭内存占用大于500MB的进程
    ```

- **练习建议**：

    1. 打开一个记事本程序，用 `TASKLIST` 找到它的进程名和PID；

    2. 用 `TASKKILL /F /IM` 关闭记事本进程；

    3. 再次打开记事本，用PID方式关闭它。

### 4. SC

- **核心作用**：Windows服务控制命令，用于显示、配置、启动、停止、安装、删除Windows系统服务。

- **基础语法**：

    ```Plain Text
    
    SC [\\计算机名] 操作 服务名 [参数]
    ```

- **核心操作**：

|    操作|    作用|
|---|---|
|    `query`|    查询服务的当前状态|
|    `queryex`|    查询服务的详细状态|
|    `start`|    启动指定服务|
|    `stop`|    停止指定服务|
|    `pause`|    暂停指定服务|
|    `continue`|    恢复暂停的服务|
|    `create`|    创建/安装新的系统服务|
|    `delete`|    删除指定的系统服务|
|    `config`|    修改服务的配置，如启动类型（自动/手动/禁用）|
- **⚠️ 警告**：修改系统服务需要管理员权限，错误操作可能导致系统功能异常！

- **实用示例**：

    ```Plain Text
    
    SC query                          # 列出所有运行中的服务
    SC query mysql                    # 查询mysql服务的状态
    SC start mysql                    # 启动mysql服务
    SC stop mysql                     # 停止mysql服务
    SC config mysql start= AUTO       # 将mysql服务设置为开机自动启动（注意=后面有空格）
    SC config mysql start= DISABLED   # 禁用mysql服务
    ```

- **练习建议**：

    1. 以管理员身份打开CMD，执行 `SC query` 查看所有服务；

    2. 查找一个你熟悉的服务（如Windows Update），查询它的状态；

    3. 查看该服务的启动类型，尝试修改为手动启动（仅测试，测试后还原）。

### 5. SHUTDOWN

- **核心作用**：允许通过本地或远程方式正确关闭、重启、休眠、注销计算机。

- **基础语法**：

    ```Plain Text
    
    SHUTDOWN [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/s`|    关闭计算机|
|    `/r`|    重启计算机|
|    `/l`|    注销当前用户|
|    `/h`|    休眠计算机|
|    `/t 秒数`|    指定关机/重启的延迟时间，单位秒，默认30秒|
|    `/f`|    强制关闭所有运行中的应用程序，不提示用户|
|    `/a`|    取消正在进行的关机/重启计划|
|    `/c "备注"`|    添加关机原因备注，会显示在关机提示中|
- **实用示例**：

    ```Plain Text
    
    SHUTDOWN /s /t 60                # 60秒后关闭计算机
    SHUTDOWN /r /t 0 /f              # 立即强制重启计算机
    SHUTDOWN /s /t 3600 /c "下班自动关机"  # 1小时后关机，添加备注
    SHUTDOWN /a                       # 取消已计划的关机/重启
    ```

- **练习建议**：

    1. 执行 `SHUTDOWN /s /t 120`，计划2分钟后关机；

    2. 执行 `SHUTDOWN /a`，取消关机计划；

    3. 尝试用 `/r` 参数计划重启，再取消。

### 7. DRIVERQUERY

## 第五部分：文本处理与过滤命令

本部分用于文本内容的搜索、过滤、排序、对比，是日志分析、代码处理的核心工具。

### 1. FIND

- **核心作用**：在一个或多个文件中搜索指定的文本字符串，是CMD基础的文本搜索工具。

- **基础语法**：

    ```Plain Text
    
    FIND [参数] "搜索内容" [文件名]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/I`|    搜索时忽略大小写|
|    `/C`|    仅显示匹配到的行数，不显示具体内容|
|    `/V`|    反向匹配，显示不包含搜索内容的行|
|    `/N`|    显示匹配行的行号|
- **实用示例**：

    ```Plain Text
    
    FIND "main" hello.py             # 在hello.py中搜索包含main的行
    FIND /I "error" 日志.txt         # 在日志文件中搜索error，忽略大小写
    FIND /C "print" *.py             # 统计所有py文件中print语句的总行数
    FIND /N "def" src\*.py           # 搜索所有py文件中的函数定义，显示行号
    DIR | FIND ".py"                 # 在dir的输出中筛选.py文件
    ```

- **练习建议**：

    1. 在你创建的py文件中，用 `FIND` 搜索包含 `print` 的行；

    2. 用 `/C` 参数统计文件中 `print` 语句的数量；

    3. 用管道符 `|` 结合 `DIR` 和 `FIND`，筛选当前目录的所有txt文件。

### 2. FINDSTR

- **核心作用**：Windows原生的强大文本搜索工具，比FIND更灵活，支持正则表达式、递归搜索、多关键词匹配，是CMD文本处理的核心工具。

- **基础语法**：

    ```Plain Text
    
    FINDSTR [参数] "搜索内容" [文件名]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/I`|    忽略大小写|
|    `/N`|    显示匹配行的行号|
|    `/C:"文本"`|    搜索包含空格的精确字符串|
|    `/R`|    使用正则表达式搜索|
|    `/S`|    在当前目录及所有子目录中递归搜索|
|    `/V`|    反向匹配，显示不包含搜索内容的行|
- **实用示例**：

    ```Plain Text
    
    FINDSTR /I /N "def " *.py        # 在所有py文件中搜索函数定义，忽略大小写，显示行号
    FINDSTR /S /I "TODO" *.py        # 递归搜索项目中所有的TODO注释
    FINDSTR /C:"server port" config.ini  # 搜索包含空格的精确字符串
    FINDSTR /R "error|warning|fail" *.log  # 正则匹配日志中的error、warning、fail
    FINDSTR /B "echo" *.bat           # 搜索以echo开头的行
    ```

- **练习建议**：

    1. 用 `FINDSTR` 递归搜索你的桌面目录中所有包含 `Python` 的txt文件；

    2. 用正则表达式搜索py文件中所有的 `import` 语句；

    3. 用 `/C` 参数搜索包含空格的字符串，对比和普通搜索的区别。

### 3. SORT

- **核心作用**：对输入的内容进行排序，支持文本文件、命令输出的排序。

- **基础语法**：

    ```Plain Text
    
    SORT [参数] [文件名]
    命令 | SORT [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    按字母升序排序|
|    `/R`|    反向排序（降序）|
|    `/N`|    按数字大小排序，而不是字符串排序|
|    `/O 输出文件`|    将排序结果写入指定文件|
- **实用示例**：

    ```Plain Text
    
    SORT 名单.txt                   # 对名单.txt的内容按升序排序
    DIR | SORT                      # 对dir的输出结果排序
    SORT /R 数字.txt                # 对数字.txt的内容降序排序
    SORT /N 成绩.txt /O 排序后的成绩.txt  # 按数字排序，结果写入新文件
    ```

- **练习建议**：

    1. 创建一个包含多个名字的txt文件，用 `SORT` 进行升序和降序排序；

    2. 创建一个包含数字的txt文件，对比 `/N` 参数和无参数排序的区别；

    3. 用 `DIR | SORT` 对目录内容排序，观察效果。

### 4. COMPACT

- **核心作用**：显示或更改NTFS分区上文件的压缩状态，实现NTFS文件系统的压缩/解压。

- **基础语法**：

    ```Plain Text
    
    COMPACT [参数] [文件名]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示当前目录文件的压缩状态|
|    `/C`|    压缩指定的文件/目录|
|    `/U`|    解压指定的文件/目录|
|    `/S`|    递归处理当前目录及子目录的所有文件|
|    `/I`|    忽略压缩过程中的错误|
- **实用示例**：

    ```Plain Text
    
    COMPACT                          # 显示当前目录文件的压缩状态
    COMPACT /C 大文件.txt            # 压缩指定文件
    COMPACT /C /S 备份目录\          # 递归压缩备份目录下的所有文件
    COMPACT /U 大文件.txt            # 解压文件
    ```

- **练习建议**：

    1. 创建一个大的txt文件，用 `COMPACT` 查看它的压缩状态；

    2. 用 `/C` 参数压缩该文件，对比压缩前后的文件大小；

    3. 用 `/U` 参数解压文件，恢复原状。

## 第六部分：网络与安全相关命令

本部分用于网络连接测试、IP配置查询、域名解析、端口查看、网络排错，是网络问题排查的核心工具。

### 1. PING

- **核心作用**：测试网络连接和延迟，向目标主机发送ICMP回显请求，判断目标是否可达、网络延迟、丢包率。

- **基础语法**：

    ```Plain Text
    
    PING [参数] 目标主机/域名/IP
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    向目标发送4个ICMP包，测试连接|
|    `-t`|    持续ping目标主机，直到按Ctrl+C停止|
|    `-n 数量`|    指定发送的ICMP包数量|
|    `-l 大小`|    指定每个ping包的大小，单位字节|
|    `-w 毫秒`|    指定超时时间，单位毫秒|
- **实用示例**：

    ```Plain Text
    
    PING baidu.com                  # 测试到百度的网络连接
    PING -t 192.168.1.1            # 持续ping网关，测试网络稳定性
    PING -n 10 baidu.com           # 发送10个包，测试平均延迟
    PING -l 1024 baidu.com         # 发送1KB大小的包，测试大包延迟
    ```

- **练习建议**：

    1. 执行 `PING baidu.com`，查看你的网络到百度的延迟和丢包率；

    2. 执行 `PING 192.168.1.1`（你的网关地址），测试内网连接；

    3. 用 `-n 20` 参数发送20个包，查看平均延迟。

### 2. IPCONFIG

- **核心作用**：查看本机所有网卡的IP配置信息，包括IP地址、子网掩码、网关、DNS服务器、MAC地址等，是网络配置查询的核心命令。

- **基础语法**：

    ```Plain Text
    
    IPCONFIG [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示所有网卡的IP地址、子网掩码、默认网关|
|    `/all`|    显示所有网卡的完整配置信息，包括MAC地址、主机名、DNS服务器、DHCP信息等|
|    `/release`|    释放所有网卡的DHCP分配的IP地址|
|    `/renew`|    重新向DHCP服务器申请IP地址|
|    `/flushdns`|    清空本地DNS解析缓存，解决域名解析异常|
|    `/displaydns`|    显示本地DNS解析缓存|
- **实用示例**：

    ```Plain Text
    
    IPCONFIG                        # 查看基础IP配置
    IPCONFIG /all                   # 查看完整的网络配置详情
    IPCONFIG /flushdns              # 清空DNS缓存，解决网页打不开的问题
    IPCONFIG /release & IPCONFIG /renew  # 重新获取IP地址，解决内网连接问题
    ```

- **练习建议**：

    1. 执行 `IPCONFIG`，查看你的电脑的IP地址、网关；

    2. 执行 `IPCONFIG /all`，查看你的网卡MAC地址、DNS服务器；

    3. 执行 `IPCONFIG /displaydns`，查看本地DNS缓存。

### 3. NSLOOKUP

- **核心作用**：查询域名对应的IP地址，以及反向IP查询域名，是域名解析排错的核心工具。

- **基础语法**：

    ```Plain Text
    
    NSLOOKUP 域名/IP [DNS服务器]
    ```

- **实用示例**：

    ```Plain Text
    
    NSLOOKUP baidu.com              # 查询baidu.com对应的IP地址
    NSLOOKUP 110.242.68.3           # 反向查询IP对应的域名
    NSLOOKUP baidu.com 8.8.8.8      # 使用谷歌DNS查询域名
    ```

- **练习建议**：

    1. 用 `NSLOOKUP` 查询 `github.com` 对应的IP地址；

    2. 用 `8.8.8.8` 和 `114.114.114.114` 两个DNS服务器分别查询同一个域名，对比结果；

    3. 尝试反向查询一个公网IP的域名。

### 4. TRACERT

- **核心作用**：跟踪数据包从本机到目标主机的路由路径，显示每一跳的IP和延迟，用于排查网络卡顿、访问异常的节点问题。

- **基础语法**：

    ```Plain Text
    
    TRACERT [参数] 目标主机/域名/IP
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    跟踪路由路径，最多30跳|
|    `-d`|    不解析IP的域名，加快跟踪速度|
|    `-h 最大跳数`|    指定最大跟踪跳数|
|    `-w 毫秒`|    指定每一跳的超时时间|
- **实用示例**：

    ```Plain Text
    
    TRACERT baidu.com               # 跟踪到百度的路由路径
    TRACERT -d baidu.com            # 不解析域名，快速跟踪路由
    TRACERT -h 10 192.168.1.1      # 最多跟踪10跳
    ```

- **练习建议**：

    1. 执行 `TRACERT baidu.com`，查看数据包到百度的路由路径；

    2. 用 `-d` 参数再次执行，对比速度差异；

    3. 观察每一跳的延迟，找到延迟最高的节点。

### 5. NETSTAT

- **核心作用**：显示所有网络连接和监听端口，以及路由表、网卡统计信息，是查看端口占用、网络连接的核心工具。

- **基础语法**：

    ```Plain Text
    
    NETSTAT [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `-an`|    显示所有连接和监听端口，以数字形式显示IP和端口，不解析域名（最常用）|
|    `-ano`|    显示所有连接、端口，以及对应的进程PID（排查端口占用必用）|
|    `-r`|    显示本机的路由表|
|    `-e`|    显示网卡的网络流量统计信息|
|    `-o`|    显示每个连接对应的进程PID|
|    `-b`|    显示每个连接对应的进程名（需要管理员权限）|
- **实用示例**：

    ```Plain Text
    
    NETSTAT -ano                    # 查看所有端口占用和对应的PID
    NETSTAT -ano | find ":8080"     # 查看8080端口被哪个进程占用
    NETSTAT -r                      # 查看本机路由表
    NETSTAT -e                      # 查看网卡流量统计
    NETSTAT -ano | find "LISTENING" # 查看所有正在监听的端口
    ```

- **练习建议**：

    1. 执行 `NETSTAT -ano`，查看你的电脑所有的网络连接；

    2. 查看3306（MySQL）、80（HTTP）、443（HTTPS）端口是否被占用；

    3. 用 `find "LISTENING"` 筛选所有正在监听的端口。

### 6. ASSOC / FTYPE

- **核心作用**：ASSOC显示或修改文件扩展名关联，FTYPE显示或修改文件扩展名关联中使用的文件类型，用于修改文件的默认打开方式。

- **基础语法**：

    ```Plain Text
    
    ASSOC .扩展名=文件类型
    FTYPE 文件类型=打开命令
    ```

- **实用示例**：

    ```Plain Text
    
    ASSOC .py                         # 查看.py文件的关联类型
    ASSOC .txt                         # 查看.txt文件的关联类型
    FTYPE Python.File                  # 查看.py文件的默认打开程序
    ASSOC .py=Python.File              # 将.py文件关联到Python.File类型
    ```

- **练习建议**：

    1. 执行 `ASSOC .py`，查看你的电脑.py文件的关联类型；

    2. 用 `FTYPE` 查看该类型对应的打开程序；

    3. 查看 `.txt` 文件的关联和打开程序。

### 7. CACLS / ICACLS

- **核心作用**：显示、修改、备份或还原文件和目录的访问控制列表(ACL)，用于管理文件/文件夹的权限。ICACLS是CACLS的增强版，功能更强大。

- **基础语法**：

    ```Plain Text
    
    ICACLS 文件/目录 [参数]
    CACLS 文件/目录 [参数]
    ```

- **核心实用示例**：

    ```Plain Text
    
    ICACLS test.txt                   # 查看test.txt的权限配置
    ICACLS test.txt /grant Users:F    # 给Users组授予完全控制权限
    ICACLS test.txt /remove Users     # 移除Users组的权限
    ICACLS 项目 /save 权限备份.acl    # 备份目录的权限配置
    ICACLS /restore 权限备份.acl      # 还原权限配置
    ```

- **练习建议**：

    1. 创建一个test.txt文件，用 `ICACLS` 查看它的权限配置；

    2. 查看 `ICACLS /?` 的帮助，了解权限配置的详细用法；

## 第七部分：磁盘与文件系统管理命令

本部分用于磁盘分区、检查、修复、格式转换、文件系统配置，是Windows磁盘管理的专业工具。

### 1. CHKDSK

- **核心作用**：检查磁盘并显示状态报告，扫描并修复磁盘错误、坏道、文件系统损坏，是修复磁盘问题的核心工具。

- **基础语法**：

    ```Plain Text
    
    CHKDSK [盘符:] [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    仅检查磁盘，不修复错误|
|    `/F`|    修复磁盘上的错误，必须锁定磁盘，磁盘正在使用时会提示下次开机时执行|
|    `/R`|    查找坏道，恢复坏道中的可读信息，包含 `/F` 的所有功能|
|    `/X`|    强制卸载磁盘卷，释放所有占用的句柄|
- **⚠️ 警告**：需要管理员权限执行，修复系统盘时需要重启电脑，中途不要中断，否则可能导致文件系统损坏！

- **实用示例**：

    ```Plain Text
    
    CHKDSK C:                        # 检查C盘，不修复
    CHKDSK D: /F                     # 检查并修复D盘的错误
    CHKDSK C: /R                     # 检查C盘，修复错误，扫描坏道
    ```

- **练习建议**：

    1. 以管理员身份打开CMD，执行 `CHKDSK D:`（非系统盘），查看磁盘状态；

    2. 查看 `CHKDSK /?` 的帮助，了解所有参数的作用；

    3. 不要随意对系统盘执行 `/F` `/R` 参数，除非你明确需要修复磁盘。

### 2. CHKNTFS

- **核心作用**：显示或修改系统启动时的磁盘检查设置，控制开机时是否对指定磁盘执行CHKDSK检查。

- **基础语法**：

    ```Plain Text
    
    CHKNTFS [参数] 盘符:
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示指定磁盘的文件系统类型|
|    `/D`|    还原默认设置，开机时所有磁盘都可能被检查|
|    `/X 盘符:`|    排除指定磁盘，开机时不执行CHKDSK检查|
|    `/C 盘符:`|    设置开机时必须检查指定磁盘|
- **实用示例**：

    ```Plain Text
    
    CHKNTFS C:                       # 查看C盘的文件系统和开机检查设置
    CHKNTFS /X C: D:                 # 开机时排除C盘和D盘的检查
    CHKNTFS /D                        # 还原默认设置
    ```

- **练习建议**：

    1. 执行 `CHKNTFS C:`，查看你的系统盘的文件系统类型；

    2. 查看 `CHKNTFS /?` 的帮助，了解更多用法。

### 3. DISKPART

- **核心作用**：Windows内置的磁盘分区管理工具，用于显示和配置磁盘分区属性，创建、删除、格式化分区，管理动态磁盘、虚拟磁盘等，是Windows磁盘管理的终极工具。

- **基础语法**：

    ```Plain Text
    
    DISKPART
    ```

    执行后会进入DISKPART交互式shell，在shell中执行分区管理命令。

- **核心常用命令**：

|    命令|    作用|
|---|---|
|    `list disk`|    列出电脑中所有的磁盘|
|    `select disk 0`|    选择第0个磁盘|
|    `list partition`|    列出选中磁盘的所有分区|
|    `select partition 1`|    选择第1个分区|
|    `detail disk`|    显示选中磁盘的详细信息|
|    `detail partition`|    显示选中分区的详细信息|
|    `exit`|    退出DISKPART shell|
- **⚠️ 高危警告**：DISKPART的错误操作会导致磁盘分区丢失、数据损坏，仅在你明确知道操作后果时使用！

- **练习建议**：

    1. 以管理员身份打开CMD，执行 `DISKPART` 进入交互式shell；

    2. 执行 `list disk`，查看你的电脑的所有磁盘；

    3. 执行 `list volume`，查看所有分区和盘符；

    4. 执行 `exit` 退出，不要执行任何修改/删除操作。

### 4. FORMAT

- **核心作用**：格式化磁盘，以便用于Windows系统，创建文件系统。

- **基础语法**：

    ```Plain Text
    
    FORMAT 盘符: [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/FS:文件系统`|    指定文件系统，支持 `NTFS`、`FAT32`、`exFAT`|
|    `/Q`|    快速格式化，仅清除文件分配表，不扫描坏道|
|    `/V:卷标`|    设置磁盘的卷标名称|
|    `/A:大小`|    指定分配单元大小|
- **⚠️ 高危警告**：格式化会清除磁盘上的所有数据，无法恢复！执行前务必确认磁盘和数据已备份！

- **实用示例**：

    ```Plain Text
    
    FORMAT D: /FS:NTFS /Q /V:数据盘  # 快速格式化D盘为NTFS格式，卷标为“数据盘”
    FORMAT E: /FS:exFAT /Q            # 快速格式化E盘为exFAT格式，适合U盘
    ```

- **练习建议**：

    1. 查看 `FORMAT /?` 的帮助，了解所有参数的作用；

    2. 不要对任何有数据的磁盘执行格式化操作，仅了解用法即可。

### 5. CONVERT

- **核心作用**：将FAT32卷转换成NTFS格式，不会损坏磁盘上的现有数据，无损转换文件系统。

- **基础语法**：

    ```Plain Text
    
    CONVERT 盘符: /FS:NTFS [参数]
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    `/FS:NTFS`|    转换为NTFS文件系统，仅支持FAT→NTFS，不支持反向转换|
|    `/V`|    详细模式，显示转换过程的所有信息|
|    `/NoSecurity`|    转换后的文件和文件夹权限对所有用户开放|
- **实用示例**：

    ```Plain Text
    
    CONVERT D: /FS:NTFS              # 将D盘从FAT32无损转换为NTFS
    CONVERT U: /FS:NTFS /V           # 转换U盘为NTFS，显示详细过程
    ```

- **练习建议**：

    1. 查看 `CONVERT /?` 的帮助，了解转换的详细用法；

    2. 转换前务必备份重要数据，避免意外情况导致数据丢失。

### 6. FSUTIL

- **核心作用**：显示或配置文件系统属性，是Windows文件系统的高级管理工具，功能非常强大。

- **基础语法**：

    ```Plain Text
    
    FSUTIL 子命令 [参数]
    ```

- **核心子命令**：

|    子命令|    作用|
|---|---|
|    `fsutil fsinfo drives`|    列出系统中所有的盘符|
|    `fsutil fsinfo drivetype 盘符:`|    查看磁盘类型（固定盘/U盘/光驱等）|
|    `fsutil volume diskfree 盘符:`|    查看磁盘的总容量、可用空间、已用空间|
|    `fsutil file createnew 文件名 大小`|    创建指定大小的空文件，用于测试|
- **实用示例**：

    ```Plain Text
    
    FSUTIL fsinfo drives             # 查看所有盘符
    FSUTIL volume diskfree C:         # 查看C盘的空间使用情况
    FSUTIL file createnew test.dat 10485760  # 创建一个10MB的空文件
    ```

- **练习建议**：

    1. 以管理员身份打开CMD，执行 `FSUTIL fsinfo drives`，查看所有盘符；

    2. 执行 `FSUTIL volume diskfree C:`，查看C盘的总容量和可用空间；

    3. 用 `createnew` 创建一个1MB的测试文件，验证大小。

### 7. LABEL / VOL

- **核心作用**：LABEL用于创建、更改或删除磁盘的卷标；VOL用于显示磁盘的卷标和序列号。

- **基础语法**：

    ```Plain Text
    
    LABEL [盘符:] [新卷标名]
    VOL [盘符:]
    ```

- **实用示例**：

    ```Plain Text
    
    VOL C:                           # 查看C盘的卷标和序列号
    LABEL D: 数据盘                  # 将D盘的卷标设置为“数据盘”
    LABEL D:                         # 删除D盘的卷标，按回车确认
    ```

- **练习建议**：

    1. 执行 `VOL C:`，查看你的系统盘的卷标和序列号；

    2. 给你的D盘设置一个新的卷标，再用 `VOL` 验证；

    3. 删除设置的卷标，恢复原状。

### 8. SUBST

- **核心作用**：将路径与驱动器号关联，把一个文件夹映射成一个虚拟磁盘盘符，方便访问深层路径。

- **基础语法**：

    ```Plain Text
    
    SUBST 盘符: 文件夹路径
    SUBST 盘符: /D
    ```

- **核心参数详解**：

|    参数|    作用|
|---|---|
|    无参数|    显示当前所有的虚拟磁盘映射|
|    `/D`|    删除指定的虚拟磁盘映射|
- **实用示例**：

    ```Plain Text
    
    SUBST Z: C:\Users\SUN\Desktop\项目  # 将项目文件夹映射为Z盘
    SUBST                                # 查看所有映射
    SUBST Z: /D                          # 删除Z盘的映射
    ```

- **练习建议**：

    1. 选择一个常用的文件夹，用 `SUBST` 映射为一个未使用的盘符（如Y:）；

    2. 打开此电脑，查看虚拟磁盘是否出现；

    3. 用 `/D` 参数删除该映射，验证是否消失。

## 第九部分：进阶学习思路

### 1. 从单条命令到批处理脚本

- 入门阶段：先熟练掌握常用的单条命令，理解每个命令的作用和参数；

- 进阶阶段：将常用的操作组合成 `.bat` 批处理脚本，实现一键自动化操作，比如：

    - 一键清理系统垃圾、Python缓存文件；

    - 一键备份项目文件到指定目录；

    - 一键配置Python开发环境、启动项目。

### 2. 深入学习批处理的高级特性

- 掌握**延迟变量扩展**，解决循环/条件中变量不更新的问题；

- 学习**函数封装**，在批处理中实现代码复用；

- 掌握**正则表达式**，结合 `FINDSTR` 实现复杂的文本处理；

- 学习**错误处理**，通过 `errorlevel` 判断命令执行结果，实现健壮的脚本。

### 3. 结合WMI/WMIC实现系统管理自动化

- 学习WMIC命令，实现系统信息采集、硬件监控、进程管理、软件安装卸载的自动化；

- 编写系统监控脚本，自动监控CPU、内存、磁盘使用率，异常时报警。

### 4. 网络自动化与排错

- 结合网络命令，编写网络监控脚本，自动ping测试网络稳定性，记录延迟和丢包率；

- 编写端口扫描脚本，检测服务器端口开放状态；

- 编写域名解析监控脚本，监控域名解析是否正常。

### 5. 过渡到PowerShell

- CMD是Windows传统的命令行工具，而PowerShell是微软新一代的命令行脚本环境，功能更强大、语法更现代、兼容性更好，支持.NET框架；

- 掌握CMD基础后，进阶学习PowerShell，是Windows系统管理、自动化运维的必然趋势；

- PowerShell完全兼容CMD命令，你之前学的CMD命令在PowerShell中几乎都能直接使用，学习成本极低。

### 6. 实战项目驱动学习

    1. 编写一个Python项目一键部署脚本，自动创建虚拟环境、安装依赖、启动项目；
