## 一、引言：为什么要学习命令行？

DOS 时代，人们只能通过输入命令与计算机交互；如今 Windows 图形界面（GUI）早已普及，命令行看似被淡忘，但从 DOS 时代的`command.com`，到 Windows XP 的`cmd.exe`，再到 Windows 7 的 PowerShell，Windows 命令行工具的功能一直在持续增强。

命令行有着 GUI 无法替代的独特价值，核心优势体现在：

1. **系统应急修复**：系统故障、病毒感染导致无法正常引导时，一张 DOS 启动盘即可进入环境完成数据备份、系统修复，且具备制作简单、体积小巧的优势。
2. **内置绿色高效工具**：系统原生自带`ping`、`netstat`、`diskpart`等大量实用工具，无需额外安装，开箱即用。
3. **深入理解系统底层**：学习命令行的过程，能更清晰地掌握 Windows 的文件管理、进程调度、网络配置等核心工作逻辑。
4. **批量自动化处理**：7-Zip、WinRAR 等大量软件都提供命令行版本，占用资源更少、运行速度更快，可通过脚本实现重复操作的全自动化。

当然命令行也存在一定门槛：界面不够直观、手动输入繁琐、学习曲线相对陡峭。本文面向 Windows 命令行零基础新手，不预设任何技术背景，覆盖从基础操作到脚本编写的全流程内容，测试环境基于 Windows XP/7，不同 Windows 版本的命令处理存在细微差异。

## 二、文件系统基础

计算机的所有程序、数据都以文件形式存在，使用计算机的核心就是对文件的管理操作，而文件系统是 Windows 管理存储设备的核心组件。

### 2.1 Windows 常见文件系统

Windows 主流的文件系统包括 FAT、FAT32、NTFS，以及用于光盘的 ISO9660（CDFS），核心差异如下：

- FAT32：兼容性强，但单个文件最大不能超过 4GB，不保留文件名大小写；
- NTFS：支持 4GB 以上大文件、文件硬链接、权限管理、大小写保留，是 Windows 系统盘的默认文件系统；
- ISO9660：专为光盘介质设计的文件系统。

### 2.2 分区与盘符

硬盘的空间划分称为**分区**，每个分区需要对应一个文件系统，决定文件在磁盘上的存储方式。

我们常说的`C:`、`D:`就是**盘符（驱动器号）**，是系统对磁盘分区的标识符号，可手动修改分配，一个分区通常仅对应一个文件系统。

### 2.3 文件命名规则

文件名称由「主文件名 + 扩展名」组成，两者用英文小数点分隔，扩展名用于标识文件的类型。

- 长度限制：Windows 中完整文件名最长支持 255 个字符（1 个中文字符计为 2 个字符）；DOS 时代遵循 8.3 规则（主文件名最长 8 字符，扩展名最长 3 字符）。
- 禁用字符：文件名不能包含以下 9 个字符：`\ / : * ? " < > |`
- 禁用名称：不能单独使用系统设备名作为文件名，包括`con`、`aux`、`com0~com9`、`lpt0~lpt9`、`nul`、`prn`。
- 大小写规则：Windows 文件名不区分大小写，`A.txt`和`a.TxT`代表同一个文件。

> 补充：Windows 默认会隐藏已知文件类型的扩展名，可通过「文件夹选项→查看」，取消勾选「隐藏已知文件类型的扩展名」，查看文件的完整名称。

### 2.4 目录的树形分级管理

如果把大量文件全部放在一个目录中，不仅查找效率极低，还容易出现误删除等操作风险。因此操作系统采用**树形分级目录结构**：

- 最顶层为**根目录**，根目录下可创建子目录，子目录可继续嵌套子目录；
- 我们可以把同一类文件放在同一个独立目录中，实现文件的分类、高效管理。

### 2.5 绝对路径与相对路径

要定位一个文件，必须明确 3 个信息：驱动器号、文件所在目录、文件名，这些信息共同组成了**路径**。路径分为绝对路径和相对路径两类：

- **绝对路径**：文件在磁盘上的完整、唯一位置，从根目录开始书写，例如`C:\WINDOWS\explorer.exe`。无任何歧义，安全性更高，适合脚本中使用。
- **相对路径**：相对于**当前工作目录**的文件位置。CMD 会始终记住一个当前目录，相对路径就以这个目录为基准。例如当前在`C:\WINDOWS`目录下，`notepad.exe`就是该文件的相对路径。

路径中还有 3 个核心特殊符号，是目录切换的高频工具：

表格

|符号|含义|
|---|---|
|`.`|代表当前目录|
|`..`|代表当前目录的上级目录（父目录）|
|`\`|代表当前盘符的根目录|

## 三、基础命令详解

### 前置基础说明

1. **打开方式**：「开始→程序→附件→命令提示符」，程序本体路径为`C:\WINDOWS\system32\cmd.exe`。
2. **命令分类**：
    
    - 内部命令：内置在 cmd.exe 中的命令，无需额外程序即可执行，例如`dir`、`copy`；
    - 外部命令：以 exe/com 可执行文件的形式存放在磁盘上，需要对应文件存在才能运行，例如`format.com`。
    
3. **执行规则**：输入命令后按回车执行，执行完成后回到命令提示符；语法错误时，CMD 会给出对应的报错提示。
4. **帮助查看**：学习任意命令的最佳方式是查看官方联机帮助，执行`help 命令名`或`命令名 /?`即可获取完整用法。
5. **通用格式**：`命令名 [参数1] [参数2] ... [操作对象1] [操作对象2] ...`，参数和操作对象的顺序通常可灵活调整，具体以帮助文档为准。

### 3.1 目录操作命令

#### 3.1.1 cd（CHDIR）：切换当前目录

**核心作用**：更改 / 显示当前工作目录

**基础格式**：`cd [/D] [盘符:][路径]`

**实用示例**：

cmd

```
cd \a1\a12\a122  # 直接进入指定的多级嵌套目录
cd \              # 一键退回当前盘符的根目录
cd ..             # 退回上一级目录
cd ..\a11         # 进入同级的a11目录
D:                # 切换到D盘（切换盘符无需cd，直接输入「盘符+冒号」即可）
cd /D D:\Project  # 跨盘符同时切换目录，/D参数强制跨盘符生效
```

#### 3.1.2 md（MKDIR）：创建目录

**核心作用**：创建新的目录（文件夹）

**基础格式**：`md [盘符:] 新建目录路径`

**实用示例**：

cmd

```
md a123           # 在当前目录创建a123子目录
md \d1            # 在C盘根目录创建d1目录
md project\src\main  # 一次性创建多级嵌套目录，无需逐级创建
```

#### 3.1.3 rd（RMDIR）：删除目录

**核心作用**：删除指定目录

**基础格式**：`rd [/S] [/Q] [盘符:]被删除目录路径`

**核心参数**：

- 无参数：仅能删除**空目录**，目录内有文件 / 子目录时会报错；
- `/S`：递归删除，删除指定目录及所有子目录、文件（可删除非空目录）；
- `/Q`：静默模式，删除时不弹出确认提示（高危参数，执行前务必确认路径）。

**实用示例**：

cmd

```
rd c13            # 删除当前目录下的空目录c13
rd /S test_folder # 删除test_folder及所有内容，会弹出确认提示
rd /S /Q test_folder # 强制静默删除，无提示，执行后数据无法恢复
```

---

### 3.2 文件操作命令

#### 3.2.1 dir：列出目录内容

**核心作用**：显示一个目录中的文件和子目录清单

**基础格式**：`dir [盘符:][路径] [参数]`

**核心高频参数**：

表格

|参数|核心作用|
|---|---|
|`/P`|分屏显示，内容满一屏后暂停，按任意键继续|
|`/W`|宽屏紧凑显示，仅显示文件名，一行展示多个|
|`/B`|极简模式，仅显示文件名 / 目录名，无日期、大小等额外信息|
|`/S`|递归显示当前目录及所有子目录的内容|
|`/A`|按属性筛选，如`/A:D`仅显示目录，`/A:-D`仅显示文件|
|`/O`|排序显示，如`/O:D`按日期升序，`/O:-D`按日期降序|

**实用示例**：

cmd

```
dir                 # 显示当前目录的所有内容
dir E:\bin /p       # 分页显示E盘bin目录的内容
dir p*.vbs          # 显示以p开头、.vbs为后缀的所有文件
dir /S /B *.py      # 递归查找当前目录及子目录下所有.py文件，仅显示完整路径
```

#### 3.2.2 copy：复制文件

**核心作用**：将一个或多个文件复制到指定位置

**基础格式**：`copy 源文件[+源文件] [目标文件]`

**实用示例**：

cmd

```
copy *.* E:\backup          # 复制当前目录所有文件到E:\backup
copy pic\ws?.jpg F:         # 复制pic目录下ws开头、后跟1个字符的jpg图片到F盘
copy a1.txt+a2.txt a_all.txt # 将a2.txt内容追加到a1.txt末尾，合并为a_all.txt
```

#### 3.2.3 del：删除文件

**核心作用**：删除一个或多个文件（仅能删除文件，无法删除目录）

**基础格式**：`del 一个或多个文件名`

**实用示例**：

cmd

```
del *.tmp       # 删除当前目录所有.tmp临时文件
del 1.txt 2.txt # 删除指定的多个文件
del /S /Q *.pyc # 递归删除当前目录及子目录下所有.pyc文件，无提示
```

#### 3.2.4 ren（RENAME）：重命名文件

**核心作用**：重命名文件 / 目录

**基础格式**：`ren 原文件名 新文件名`

**实用示例**：

cmd

```
ren old.txt new.txt  # 将old.txt重命名为new.txt
ren *.txt *.md       # 批量将所有txt文件重命名为md文件
```

### 3.3 通配符

通配符是命令行批量操作的核心，用于模糊匹配文件名，Windows 支持两种核心通配符：

表格

|通配符|作用|示例|匹配结果|
|---|---|---|---|
|`?`|匹配**任意单个字符**|`help?`|help1、help2、help3|
|`*`|匹配**零个或多个任意字符**|`help*`|help、help1、help123|
|`*.*`|匹配所有文件|`del *.*`|删除当前目录所有文件|

### 3.4 命令输入高效技巧

#### 3.4.1 命令修改与历史

- 未回车时：用左右方向键移动光标修改错误内容，无需删除重打；
- 已回车执行：用**上 / 下方向键**快速调出历史执行过的命令，无需重复输入；
- 核心快捷键速查：
    
    表格
    
    |快捷键|核心作用|
    |---|---|
    |F1|逐字符重复上一条命令|
    |F3|完整复制上一条命令|
    |Tab|自动补全文件名 / 目录名（最常用，大幅提升输入效率）|
    |Ctrl+C|强制终止当前命令 / 程序的执行|
    |Esc|作废当前输入行|
    

#### 3.4.2 复制粘贴

CMD 窗口支持右键「标记」选中内容，回车复制，右键「粘贴」，可快速复用已有命令的部分内容，减少重复输入。

## 四、进阶核心功能

### 4.1 输入输出重定向与管道

默认情况下，命令的输入来自键盘（标准输入），输出到屏幕（标准输出）；而重定向功能可以修改输入输出的来源 / 目的地，管道符可以实现多个命令的联动，是命令行进阶的核心。

#### 4.1.1 输出重定向

表格

|符号|核心作用|实用示例|
|---|---|---|
|`>`|覆盖写入：将命令输出写入文件，文件不存在则新建，已存在则完全覆盖|`dir > dir.txt` # 将 dir 的输出保存到 dir.txt|
|`>>`|追加写入：将命令输出追加到文件末尾，不覆盖原有内容|`dir >> dir.txt` # 将 dir 的输出追加到 dir.txt 末尾|

#### 4.1.2 输入重定向

`<` 符号：将文件内容作为命令的输入，替代键盘输入。

示例：

cmd

```
sort < names.txt > names-sorted.txt
# 读取names.txt的内容，排序后写入names-sorted.txt
```

#### 4.1.3 管道符 `|`

管道符的核心作用是：**将前一个命令的输出，直接作为后一个命令的输入**，实现命令的无缝联动，是命令行高效操作的灵魂。

经典实用场景：

1. 分屏显示长内容：
    
    cmd
    
    ```
    sort /? | more  # 分屏查看sort命令的帮助，一屏显示不完时自动暂停
    ```
    
2. 过滤输出内容：
    
    cmd
    
    ```
    set | find "Path"  # 查看所有环境变量，仅筛选出包含Path的行
    ```

### 4.2 系统特殊设备

Windows 中可通过设备名直接操作硬件 / 系统组件，实现很多特殊功能，核心设备名如下：

表格

|设备名|含义|实用示例|
|---|---|---|
|`CON`|控制台（键盘 + 屏幕）|`copy con t.txt` # 读取键盘输入，直接写入 t.txt，按 Ctrl+Z 结束输入|
|`PRN`|默认打印机|`print test.txt > PRN` # 打印文本文件|
|`NUL`|空设备（系统黑洞）|`copy nul t.txt` # 将空内容写入 t.txt，清空文件大小，实现文件粉碎|
|`COM1~COM4`|串行端口|-|
|`LPT1~LPT3`|并行端口|-|

### 4.3 环境变量

环境变量是「变量名 = 变量值」的键值对，用于存储系统配置、动态参数，在命令和脚本中通过`%变量名%`的格式引用其值。

#### 4.3.1 基础操作

cmd

```
set               # 列出当前所有环境变量及其值
set str=Hello!    # 创建/修改变量，变量名str，值为Hello!
echo %str%        # 引用变量，输出变量的值
cd %windir%       # 引用系统内置变量windir，等价于cd C:\WINDOWS
```

#### 4.3.2 核心系统变量 PATH

PATH 是特殊的环境变量，存储了一组用分号`;`分隔的目录路径。CMD 执行命令时，会按 PATH 中的目录顺序查找对应的可执行文件（exe/com）。

**实用场景**：

比如 wget.exe 位于`D:\GnuWin32\bin`，默认情况下直接输入 wget 会报错「不是内部或外部命令」，将目录加入 PATH 后，即可直接用文件名执行：

cmd

```
# 临时生效（仅当前CMD窗口有效，关闭后失效）
set PATH=%PATH%;D:\GnuWin32\bin

# 永久生效
# 计算机→右键→属性→高级系统设置→环境变量→系统变量→编辑PATH，添加对应目录
```

## 五、批处理文件简介

### 5.1 什么是批处理文件

批处理文件是纯文本文件，包含一条或多条 CMD 命令，文件扩展名为`.bat`或`.cmd`。在 CMD 中输入批处理文件名，cmd.exe 会按顺序执行文件中的所有命令。

- 核心优势：一次编写，多次运行，完美解决重复性、批量性的操作；
- 进阶能力：支持`for`、`if`、`goto`等流程控制命令，可实现复杂的逻辑判断和循环。

### 5.2 批处理核心常用命令

表格

|命令|核心功能|
|---|---|
|`rem`|添加注释，CMD 执行时会完全忽略该行内容|
|`@echo off`|关闭整个脚本的命令回显，仅显示输出结果，是批处理脚本的标配开头|
|`if`|批处理中的条件判断，实现分支逻辑|
|`for`|循环命令，为一组文件 / 数据重复执行指定命令|
|`goto`|跳转到脚本中指定的标签行，实现循环 / 分支跳转|
|`pause`|暂停脚本执行，显示「请按任意键继续...」，避免脚本执行完窗口一闪而过|
|`call`|从当前脚本调用另一个批处理脚本，执行完后返回原脚本继续执行|
|`title`|设置 CMD 会话的窗口标题|
|`start`|启动新窗口运行指定的程序 / 命令|

## 六、实战应用案例

### 6.1 批量重命名文件

**场景**：下载的图片文件名不规范，需要统一重命名为 01.jpg~20.jpg

cmd

```
# 切换到图片所在目录
D:
cd D:\pics

# 批量修改001.jpg~009.jpg为01.jpg~09.jpg
for %i in (01,02,03,04,05,06,07,08,09) do ren 0%i.jpg %i.jpg

# 批量修改010.jpg~020.jpg为10.jpg~20.jpg
for /L %i in (10,1,20) do ren 0%i.jpg %i.jpg
```

### 6.2 批量粉碎文件

**场景**：彻底删除文件，避免被数据恢复软件找回

cmd

```
# 先清空文件内容，再删除，恢复后仅能得到空文件
for %i in (*.jpg) do copy nul %i /y
del *.jpg /q
```

### 6.3 一键清理 Windows 临时文件夹

**场景**：定期清理系统临时文件，释放磁盘空间

创建`CleanTmp.cmd`脚本，内容如下：

cmd

```
@echo off
rem 清理Windows临时文件夹脚本
echo 正在清理临时文件...
rd /s /q %tmp%
md %tmp%
echo 清理完成！
pause
```

> `%tmp%`是系统内置变量，指向当前用户的临时文件夹，双击脚本即可一键清理。

### 6.4 一键切换 IP/DNS 配置

**场景**：笔记本在家和公司往返，需要频繁切换 DHCP 和静态 IP

1. 自动获取 IP 脚本`dhcp.cmd`：
    
    cmd
    
    ```
    set name="本地连接"
    netsh interface ip set address %name% dhcp
    netsh interface ip set dns %name% dhcp
    pause
    ```
    
2. 静态 IP 脚本`static_ip.cmd`：
    
    cmd
    
    ```
    set name="本地连接"
    netsh interface ip set address %name% static 192.168.3.19 255.255.255.0 192.168.3.1 0
    netsh interface ip set dns %name% static 192.168.3.1
    netsh interface ip add dns %name% 8.8.8.8
    pause
    ```
    

> 双击对应脚本即可一键切换 IP 配置，无需手动打开网络设置。

### 6.5 批量下载系列图片

**场景**：下载地址有规律的系列图片（如 1.jpg~10.jpg），需提前安装 GNU Wget 并加入 PATH。

cmd

```
# 循环下载1.jpg到10.jpg，多进程并发下载
for /L %i in (1,1,10) do start wget http://www.example.com/%i.jpg
```

### 6.6 平坦复制（合并子目录文件）

**场景**：将一个目录下所有子目录的文件，全部复制到同一个目标目录，不保留原有的子目录结构。

cmd

```
# 源目录SRC，目标目录DEST
for /R "%SRC%" %i in (*) do copy "%i" "%DEST%"
```

## 七、常见问题解答

### Q1：echo 命令输出总会自动换行，如何输出不换行的字符串？

A：可使用 set 命令的特殊用法：

cmd

```
set/p=要输出的内容<nul
```

### Q2：如何创建资源管理器中无法正常删除的特殊目录？

A：用 md 命令创建带`..\`的目录：

cmd

```
md test..\  # 创建名为test..的特殊目录
rd test..\  # 删除该特殊目录
```
